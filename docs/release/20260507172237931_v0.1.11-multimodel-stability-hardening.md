# v0.1.11 Multimodel Stability Hardening

## Root Cause

- Rapid location switching could leave timed-out multimodel insight and distribution fetches running underneath the UI timeout.
- `MULTIMODEL_ORIGIN_UNAVAILABLE` was still eligible for client-side retry paths, amplifying origin or Worker failures during fast switching.
- Origin location-version skew on multimodel routes could be counted as a global origin failure and open the multimodel circuit for unrelated cities.
- Dashboard source metadata could report multimodel as fresh when both analysis and image payloads were unavailable.

## Changes

- Linked multimodel insight and distribution requests to abort controllers so client timeout cancels the underlying request chain.
- Removed origin-unavailable from multimodel warmup retry and blocked generic API retry for `MULTIMODEL_ORIGIN_UNAVAILABLE`.
- Classified multimodel origin location-version skew as non-retryable and prevented it from opening the global multimodel circuit.
- Marked source metadata multimodel freshness as `fallback_error` when both multimodel analysis and image are unavailable.
