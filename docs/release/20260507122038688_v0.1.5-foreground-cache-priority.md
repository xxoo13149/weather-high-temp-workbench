# v0.1.5 foreground cache priority

## Changes
- Allow a cold foreground cache read to overtake an in-flight background load.
- Prevent dashboard/status background warmups from making analysis users wait behind slower background multimodel refreshes.
- Keep old background results from overwriting the foreground result once the foreground load wins.

## Why
- Online smoke found a cold Wuhan analysis path where `insights` returned quickly but `distribution` could sit behind a background refresh long enough for the browser to hit the 20s client timeout.
