# v0.1.7 Defer Distribution Load

## Goal

Reduce foreground multimodel pressure during rapid city switching after the v0.1.6 production measurement showed slot saturation from still-starting insight and distribution together.

## Changes

- Start distribution only after insight succeeds and the request is still the current city/timestamp.
- Remove distribution from location surface prewarm; prewarm now avoids spending Worker capacity on data that is not immediately visible.
- Preserve the v0.1.6 progressive rendering behavior: insight is committed first, distribution fills in later.

## Validation

- `npm run check`
- `npm run build`
- `npm test`
- Targeted regression: `npm test -- tests/app.test.ts tests/frontend-api.test.ts tests/meteoblue-service.test.ts`

## Production Finding That Triggered This Patch

The v0.1.6 online all-city insight measurement improved visible rendering but still produced `MULTIMODEL_CACHE_LOAD_BUSY` for some cities after rapid online smoke and all-city probing. The remaining pressure source was client-side distribution work that still started too early.
