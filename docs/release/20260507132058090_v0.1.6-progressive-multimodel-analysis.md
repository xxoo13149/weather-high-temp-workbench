# v0.1.6 Progressive Multimodel Analysis Rendering

## Goal

Make rapid city switching resilient when Meteoblue multimodel distribution data is slow, overloaded, or temporarily unavailable.

## Changes

- Render analysis insight as soon as the insight request succeeds; distribution now fills in later and no longer blocks ranked model display.
- Keep distribution failures local to distribution cards instead of blanking the full analysis workspace.
- Stop cold `getMultiModelStatus` calls from kicking off hidden multimodel page/highcharts loads.
- Reduce background multimodel cache load concurrency from 7 to 2 so foreground user requests keep capacity during rapid switching.
- Update regression tests around cold status, image-only status, poisoned cached status, and expired-cache background refresh limits.

## Validation

- `npm run check`
- `npm run build`
- `npm test`
- Targeted regression: `npm test -- tests/meteoblue-service.test.ts tests/cache.test.ts tests/cloudflare-worker.test.ts tests/app.test.ts tests/frontend-api.test.ts`

## Operational Notes

- Cold status responses may report analysis as `unavailable` until a foreground insight/distribution request loads data.
- Existing expired distribution cache entries still refresh in the background, but at a lower concurrency ceiling.
- This version is intended to deploy after v0.1.5, which still allowed distribution latency to delay the visible analysis result.
