# v0.1.9 Status Based Multimodel Retry

## Goal

Retry code-less transient 502/503/504 multimodel failures in the same multimodel retry loop as coded Meteoblue failures.

## Changes

- Add status-based retry handling for multimodel `WeatherApiError` responses with HTTP 502, 503, or 504.
- Keep the existing retryable payload check so explicit non-retryable responses still fail fast.

## Validation

- `npm run check`
- `npm run build`
- `npm test -- tests/app.test.ts tests/frontend-api.test.ts`
- `npm test -- tests/app.test.ts tests/frontend-api.test.ts tests/meteoblue-service.test.ts`

## Production Finding That Triggered This Patch

The v0.1.8 frontend-equivalent 52-city run still had code-less 503 responses that did not enter the outer multimodel retry loop. This patch makes those transient statuses retryable even when the response body is not a standard JSON error payload.
