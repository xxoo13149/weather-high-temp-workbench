# v0.1.10 - Multimodel Origin Proxy

## Why

Production analysis-all testing on Cloudflare showed 43/52 successful multimodel insight reads. The 9 failures were Cloudflare 1102 responses (`Worker exceeded resource limits`), which means the edge Worker was still doing too much heavy multimodel scraping/parsing work under city-switch pressure.

## Changes

- Proxy `/api/weather/multimodel/status`, `/distribution`, `/insights`, and `/image` from Cloudflare to the origin service when `KELLY_SERVER_BASE_URL` is configured.
- Return a retryable `MULTIMODEL_ORIGIN_UNAVAILABLE` response when the multimodel origin is unhealthy instead of running heavy local fallback work inside the Worker.
- Add multimodel origin timeout and circuit health fields to `/healthz`.
- Stop marking cold multimodel status as fake `revalidating` when no cache load is active.
- Extend the frontend analysis timeout budget to 35s and retry code-less 502/503/504 responses, including Cloudflare 1102-style responses.
- Show insight errors in the empty model list area instead of leaving the user with a vague empty state.

## Verification

- `npm run check`
- `npm run build`
- `npm test`
- Targeted Worker and Meteoblue tests: 71 passed

## Production Note

This release is intended to remove the Cloudflare Worker as a bottleneck for long-running multimodel reads. Origin capacity and stale-data fallback should be the next optimization layer if any origin-side 503s remain.
