# 接口与显示契约加固计划

## 背景
- 当前问题不是单点乱码，而是前后端在“文本来源、翻译职责、显示兜底、编码校验”上的职责边界不够稳定。
- 用户要求在不破坏现有效果、交互和功能的前提下，做可维护的统一接口与显示契约修复，避免同类问题反复出现。

## 本轮目标
- 统一天气中文显示来源，避免前端重复翻译、重复映射、重复兜底。
- 清理分析工作区残留乱码与混杂文案，统一消费稳定字段。
- 补齐测试、编码与构建侧的最小防回归措施。
- 保持现有路由、布局、交互和已正常工作的视觉效果不回退。

## 执行分工
- 主线程：整合方案、实施跨层改动、做最终验证。
- Hubble：诊断前端乱码与显示契约根因。
- Ampere：修复 `zip/src` 前端显示链路。
- Volta：修复 `src` 下中文源文本与后端兜底链路。
- Pascal：审查测试、构建与编码防线缺口。

## 预期产出
- 前端统一显示文本辅助层与类型契约收敛。
- 后端统一中文产出/清洗与安全兜底。
- 测试配置避免扫描快照目录，减少误报与重复执行。
- 完整验证记录与 release 文档。
