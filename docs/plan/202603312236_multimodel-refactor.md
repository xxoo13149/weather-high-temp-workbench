# 多模型洞察与原图卡片重构计划

- 时间: 2026-03-31 22:36 Asia/Shanghai
- 变更编号: PLAN-20260331-01
- 状态: Implemented

## 问题背景

- 多模型时间轴存在固定 `+8h` 偏移风险。
- 首页洞察卡信息过载，原图缩略图没有阅读价值。
- 项目缺少最小化变更记录，后续不利于回退。

## 实施范围

1. 后端修正 `multimodel` 时间解析与洞察排序口径。
2. 前端改成“24 小时预测 + 原图信息卡 + 单模型洞察卡 + 详情抽屉”。
3. 初始化 `docs/plan`、`docs/release`、`docs/decision` 记录体系。

## 验收条件

1. `point.name` 可用时不得再出现固定 `+8h` 偏移。
2. 首页只展示一个最接近模型结论。
3. 原图首页不再展示缩略图。
4. `npm run test`、`npm run build:server`、`npm run build:web`、`npm run check` 通过。

## 回退方式

1. 先恢复旧前端入口。
2. 再恢复旧 `insights` 响应契约。
3. 删除新增文档即可回到实施前的治理状态。
